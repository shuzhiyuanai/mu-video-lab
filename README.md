# 前端面试题

本项目包含三道前端面试题，分别考察 **流式文本与竞态处理**、**视频播放与进度控制** 和 **动效设计与 UI 审美**。

## 快速开始

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

浏览器打开 [http://localhost:3000](http://localhost:3000)，在首页点击对应按钮即可进入各题页面。

## 题目一：AI 提示词优化器 (Streaming Text)

**场景**：用户输入简单描述，前端调用 Mock 接口显示 AI 优化后的 Prompt。

### 核心要求

1. **打字机效果**：点击提交后，必须实现打字机效果（逐字显示内容）。
2. **竞态处理**：如果用户在第一个回复没打完时又点了「生成」，代码是否会崩溃或出现文字重叠？需正确处理竞态。
3. **状态管理**：区分「正在思考」和「正在输出」两种 UI 状态。

### 实现入口

- 页面路由：`/prompt-optimizer`
- 页面文件：`app/prompt-optimizer/page.tsx`

### Mock 接口

- **POST** `/api/optimize-prompt`
- **请求体**：`{ "prompt": "用户输入的描述" }`（API 不做实际处理，但接口保持完整，便于面试者对接）
- **响应**：`text/plain` 流式返回，逐字推送
- **行为**：
  1. 先等待约 1.5 秒模拟「思考」阶段（此时无数据推送）
  2. 再逐字推送文案，每字间隔约 80ms
- **返回内容**：一段约 300 字的小故事（含换行分段）

> **提示**：前端使用 `fetch` + `response.body.getReader()` 消费流即可。
> 思考阶段的 1.5s 内不会收到任何 chunk，面试者需要据此区分两种 UI 状态。

## 题目二：视频剪辑台 (Video Montage)

**场景**：模拟一个简易视频剪辑台。页面分为三个区域：**左侧素材列表**、**右侧预览区**、**底部时间线轨道**。用户从素材列表将视频添加到时间线，设置每个片段的截取区间（如 A 视频 2–5 秒），最后点击播放，前端按时间线顺序拼接播放。

> 项目已提供骨架页面（三栏布局、素材列表、类型定义），面试者需要实现核心播放与交互逻辑。

### 核心要求

1. **素材添加**：从左侧素材面板点击「添加到时间线」按钮，将视频片段添加到底部时间线轨道中。
2. **截取区间设置**：每个已添加到时间线的片段可设置起止时间（秒），例如 A 视频截取 2s–5s、B 视频截取 1s–3s。
3. **顺序拼接播放**：点击播放后，按时间线中的顺序依次播放各片段对应的截取区间，通过 `<video>` 的 `currentTime` + `onTimeUpdate` 实现无缝、无黑屏切换（纯前端逻辑，不要求后端合并视频文件）。
4. **进度条**：底部有一个进度条，显示当前播放到「总拼接时长」的哪个位置。

### 加分项

- **拖拽交互**：素材可通过 HTML5 Drag & Drop 拖入时间线，时间线内片段可拖拽排序；
- **时间线可视化**：时间线上的片段以色块形式展示，宽度按时长比例缩放，可直观看到各片段占比；
- **Canvas 转场 / 预加载策略**：使用 Canvas 做转场渲染，或实现视频预加载以消除切换时的加载延迟；
- **片段管理**：支持删除已添加的片段、同一素材可重复添加多次。

### 考察维度

| 维度 | 说明 |
|------|------|
| **组件状态设计** | 时间线片段列表的数据结构设计，播放状态机的管理 |
| **DOM 事件处理** | Drag & Drop API 或 Pointer Events 的运用（加分项） |
| **视频 API 掌握** | `currentTime` / `timeupdate` / `ended` 等 API 的精细控制 |
| **UI/UX 完成度** | 时间线是否直观易用，交互反馈是否到位 |

### 实现入口

- 页面路由：`/video-montage`
- 页面文件：`app/video-montage/page.tsx`
- 骨架页面已搭好布局，核心逻辑处有 `// TODO` 标注

### 视频资源

项目已在 `public/videos/` 目录下提供两个视频文件：

| 文件 | 访问路径 | 说明 |
|------|---------|------|
| `a.mp4` | `/videos/a.mp4` | 视频片段 A |
| `b.mp4` | `/videos/b.mp4` | 视频片段 B |

> 前端直接通过 `/videos/a.mp4`、`/videos/b.mp4` 即可引用。

## 题目三：AI 产品首页 Hero 区 (Creative Landing)

**场景**：在没有 UI 设计稿的前提下，独立完成一个 AI 产品首页的 Hero 区域。考察候选人的 **视觉审美**、**动效能力** 和 **创意表达**。

### 核心要求

1. **视觉风格**：整体呈现「未来感 / 科技感」，配色、排版、层次感需体现个人审美水平。
2. **入场动画**：页面加载后，至少有一个元素具备 entrance 动画（如淡入、上移、缩放等），过渡自然流畅。
3. **按钮交互**：Hero 区至少包含一个 CTA 按钮，hover 时必须有明确的视觉反馈（缩放、光晕、渐变偏移等均可，不可无反馈）。
4. **性能要求**：所有动效需流畅运行，不出现明显掉帧或卡顿。

### 加分项

- **粒子 / Canvas 动效**：使用 Canvas 或 WebGL 实现背景粒子、星空、网格等动态效果；
- **渐变流动 / 光效**：背景或元素上有流动的渐变色、光线扫过、辉光脉冲等视觉效果；
- **响应式适配**：在不同屏幕尺寸下布局和动效均表现良好；
- **微交互细节**：鼠标跟随、视差滚动、元素间联动等增强沉浸感的细节。

### 考察维度

| 维度 | 说明 |
|------|------|
| **UI 审美** | 没有设计稿时能否自主产出高质量视觉效果（这是最关键的考察点） |
| **动效实现** | CSS Animation / Transition / Canvas / requestAnimationFrame 的运用能力 |
| **代码组织** | 动效逻辑是否与业务组件合理分离，是否易维护 |
| **创意表达** | 是否有令人眼前一亮的创意，而非套模板 |

### 实现入口

- 页面路由：`/ai-hero`
- 页面文件：`app/ai-hero/page.tsx`

> **提示**：本题不限定使用任何第三方动效库（如 Framer Motion、Three.js、tsParticles 等），也可以纯 CSS / Canvas 手写。重点不在于用了什么工具，而在于最终呈现效果。

## 项目结构

```
app/
  page.tsx                        # 首页，三道题的入口
  api/
    optimize-prompt/
      route.ts                    # Mock API：chunk stream 返回优化文案（题目一）
  prompt-optimizer/
    page.tsx                      # 题目一：AI 提示词优化器
  video-montage/
    page.tsx                      # 题目二：双源视频剪辑拼接器
  ai-hero/
    page.tsx                      # 题目三：AI 产品首页 Hero 区
public/
  videos/
    a.mp4                         # 视频片段 A（题目二）
    b.mp4                         # 视频片段 B（题目二）
```
